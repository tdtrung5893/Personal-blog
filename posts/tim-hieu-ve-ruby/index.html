Tìm hiểu về Ruby

Tran Duc Trung 15/07/2018

    1) Các đặc điểm cơ bản của lập trình hướng đối tượng. Chúng được thể hiện như thế nào trong Ruby

    Lập trình hướng đối tượng có 4 đặc điểm cơ bản:

    - Tính trừu tượng (Abstraction): Là trừu tượng hóa và định nghĩa 1 cách khái quát các tính chất, thuộc tính của 1 đối tượng. Trong ruby thì tính trừu tượng được thể hiện thông qua lớp hay còn gọi là Class. Trong Class sẽ bao gồm có thuộc tính - là các biến (local, global, instance, class) và phương thức - là các method

    - Tính đóng gói (Encapsulation): Là để giấu đi các phương thức của đối tượng đối với bên ngoài, tức là chỉ có developer mới có thể can thiệp và chỉnh sửa được vào các phương thức này. Trong ruby thì tính đóng gói được thể hiện qua Public, Protect, Private và Attr accessor, GetSet

    - Tính kế thừa (inheritance): 1 Class có thể kế thừa và sử dụng các phương thức đã được định nghĩa của 1 lớp khác (hay còn gọi là lớp cha). Trong ruby thì tính kế thừa được thể hiện bằng dấu nhỏ hơn "<". Ví dụ Class Hyundai < Class Car (SuperClass)

    - Tính đa hình (Polymorphism): Cùng 1 phương thức và các Class khác nhau cùng sử dụng sẽ cho ra kết quả khác nhau. Trong ruby thì tính đa hình được thể hiện bằng cách định nghĩa các phương thức trong module và các Class sẽ nhúng module vào để sử dụng các phương thức đó

    2) Lấy một ví dụ về tính đa hình? ví dụ về tính kế thừa? ví dụ về tính đóng gói? ví dụ về tính trừu tượng?

    - Tính đa hình:

         Module Speak
           def sound(animal)
             puts "#{animal}"
           end
         end
         def Cat
           include Speak
         end
         def Lion

           include Speak
         end
         cat = Cat.new
         cat.sound(meow) => #meow
         lion = Lion.new
         lion.sound(Grrrr) => #Grrrr

    - Tính trừu tượng:

         class Cat
           def initialize
              puts "Meow"
           end
         end
         Cat.new => Meow

    - Tính đóng gói:

        class Animal
          def initialize
            lion
          end
          private
          def lion
            puts "Grrr"
          end
          a = Animal.new   # a.lion => Grrr
        class Animal
          def initialize
            self.lion
          end
          protected
          def lion
            puts "Grrr"
          end
        a = Animal.new   # a.lion => Grrr

    - Tính thừa kế:

         class Animal
            def initialize
               puts "Created animal"
            end
         end
         class Cat < Animal
            def initialize
               super
               puts "Meow"
            end
         end
         Cat.new
         # Created animal
         # Meow

    3) Thế nào là accessor và setter trong Ruby? Phân biệt các từ khoá public, private và protecte?

    - Setter và Getter: getter và setter dùng để truy xuất dữ liệu và chỉnh sửa chúng. Trong Ruby có 2 phương thức để làm công việc trên là attr_writer (setter) và atter_reader (getter)

         class Animal
            def getter_cat
               @sound
            end
            def setter_cat(sound)
               @sound = sound
            end
         end
         cat = Animal.new
         cat.setter_cat = "Meow"
         cat.getter_cat    #Meow

    - Accessor: Trong ruby thì attr_accessor sẽ tạo ra cả 2 phương thức attr_reader và attr_writer, đồng thời tạo ra luôn biến instance cho đối tượng

         class Animal
            attr_accessor :sound
            def speak
                puts "Cat speak #{sound}"
            end
         end
         cat = Animal.new
         cat.sound = "Meow"
         cat.speak    #Meow

    - Phân biệt Public, Private, Protected:

    • + Public: Mặc định các phương thức trong Ruby là Public, tức là bên ngoài có thể truy xuất được.
    • + Private: Các phương thức là private thì chỉ có thể truy xuất được bên trong Class của đối tượng còn bên ngoài thì sẽ không thể truy xuất được.
    • + Protected: Cũng giống như là private tuy nhiên các phương thức protected có thể truy xuất được bằng từ khóa self còn private thì không.

    4) Overriding và Overloading?

    - Overloading: Là phương thức dùng để ghi chồng dữ liệu. Việc ghi chồng dữ liệu này được thực hiện với 2 phương thức có cùng tên và khác đối số

         class Animal
            def cat(a)
                puts "#{a}"
            end
            def cat(a, b)
               puts "#{a} + #{b}"
            end
         end
         cat = Animal.new
         a.cat("Meow", "meoww")
         # Meow + meoww

    - Overriding: Là phương thức dùng để ghi đè dữ liệu. Việc ghi đè dữ liệu này được thực hiện ở 2 class có quan hệ cha con với nhau (tính kế thừa) với 2 phương thức cùng tên và cùng đối số

         class Animal
            def speak(sound)
                puts "#{sound}"
            end
         end
         class Cat < Animal
            def speak(sound)
               puts "speak #{sound}"
            end
         end
         c = Cat.new
         c.speak("Meow")
         # speak Meow

    5) Composition vs Inheritance

    - Inheritance: Là tính chất thừa kế được dùng trong thiết kế lập trình hướng đối tượng. Tuy nhiên, nếu như sử dụng không hợp lý sẽ rất dễ dẫn đến trường hợp bị lặp code và khó bảo trì sau này. Ví dụ:

         class Vehicle
            def run
                refill
                load
            end
         end

         class Car < Vehicle
            def load
            end
         end

         class Truck < Vehicle
            def load
            end
         end
         
         class PetrolCar < Car
            def refill
               puts "refill with fuel"
            end
         end

         class ElectricCar < Car
            def refill
               puts "refill with electricity"
            end
         end

         class PetrolTruck < Truck
            def refill
               puts "refill with fuel"   --> Code repeat
            end
         end

         class ElectricTruck < Truck
            def refill
               puts "refill with electricity"   --> Code repeat
            end
         end

    - Composition: Được tạm dịch ra là tổng hợp code, là 1 kỹ thuật dùng để giải quyết vấn đề của Inheritance. Chúng ta sẽ gom tất cả các phương thức cần tạo vào 1 đối tượng riêng được tạo bởi 1 class mới, sau đó ta sẽ tham chiếu các đối tượng của ta đến các đối tượng mới tạo. Ví dụ:

         class PetrolEngine
            def refill
                puts "refill with fuel"
            end
         end

         class ElectricEngine
            def refill
                puts "refill with electricity"
            end
         end

         class BodyCar
            def load
                puts "load passengers"
            end
         end
         
         class BodyTruck
            def load
               puts "load cargo"
            end
         end

         class Vehicle
            def initialize(engine, body)
               @engine = engine
               @body = body
            end
            def run
               @engine.refill
               @body.load
            end
         end

         petrol_car = Vehicle.new(engine: PetrolEngine.new, body: BodyCar.new)
         petrol_car.run
         # refill with fuel
         # load passengers

    Kết luận: Kỹ thuật composition sẽ làm cho code dài hơn nhưng lại clear hơn và dễ dàng thay đổi, bảo trì sau này. Ngoài ra nó còn làm linh hoạt hơn trong cách sử dụng đối với các class mới tạo.
