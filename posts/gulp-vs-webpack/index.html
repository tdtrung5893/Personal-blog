Gulp vs Webpack?

Tran Duc Trung 24/07/2018

    Gulp là gì?

          Gulp là 1 công cụ giúp bạn tự động hóa nhiều task (nhiệm vụ) trong quá trình phát triển web. Nó thường được sử dụng để làm các tác vụ front end như:

    • + Tạo ra 1 web server
    • + Auto reload server khi lưu thay đổi 1 file nào đó
    • + Sử dụng các Preprocessor giống như SASS, LESS, Stylus...
    • + Tối ưu hóa các file CSS, JS, Image...

          Về cơ bản thì gulp chia ra làm các task để làm việc, để chạy mỗi task thì ta chỉ cần gõ gulp tên-task. Ví dụ như muốn chạy task js thì chỉ cần gõgulp js là được.

          Để sử dụng gulp thì đầu tiên ta tải package gulp bằng lệnh npm i gulp --dev sau đó tạo 1 file dùng để config tên là gulpfile.js

    -> Ta khai báo thư viện gulp bằng cách:var gulp = require('gulp');

          Tương tự để sử dụng những tính năng khác của gulp như là auto reload server, convert các file SASS sang CSS, tối ưu hóa các file CSS, JS, Image thì ta cũng cần phải khai báo tương tự

    - Ví dụ để auto reload server thì ta sử dụng package browser-sync

         Khai báo trong gulpfile.js

     var browserSync = require('browser-sync').create();
     gulp.task('browserSync', function() {
       browserSync.init({
         server: {
           baseDir: 'app'
         },
       })
     });

         Giờ ta muốn khi có 1 file nào đó được chỉnh sửa thì server sẽ tự động reload thì ta thêm đoạn code sau:

     gulp.task('watch', ['browserSync'], function(){
       gulp.watch('app/index.html', browserSync.reload);
       gulp.watch('app/js/*.js', browserSync.reload);
     });

     - Trên command line ta sẽ chạy lệnhgulp watch

     -> Ở đây ta dùnggulp.watch để theo dõi tất cả những file có đuôi là js trong thư mục app/js và file index.html, bất cứ khi nào các file đó có sự thay đổi thì server sẽ tự động reload

         Tóm lại Gulp là công cụ tự động hóa tốt, cho phép chúng ta setup một mớ các tiến trình phức tạp cần thực hiện (một cách tự động) bằng javascript, rồi chạy nó bằng một lệnh duy nhất.

    Webpack là gì?

         Webpack là 1 công cụ dùng để quản lý phần code client-side. Hay nói cách khác Webpack là 1 module bundler, nó nhận vào các module cùng với các dependencies và generate ra các static assets tương ứng.

         Đầu tiên để cài đặt webpack, ta chạy lệnh sau:npm install webpack --save-dev

         Ví dụ ta có 2 file js và 1 file index.html

       //hello.js
       export default function hello(name){ console.log('Hello '+ name); }

       //index.js
       improt 'hello' from './hello';
       hello('Trung');

       //index.html
       <html>
       ...
       <script src="../../bundle.js"></script>
       ...
       </html>

         Ta tiến hành tạo 1 file để config tên là webpack.config.js với nội dung như sau:

       module.exports = {
         entry: './index.js',
         output: { filename: 'bundle.js' }
       }

         Khi ta chạy webpack trên command line và sau khi chạy file index.html ta sẽ thấy xuất hiện dòng chữ: Hello Trung

         Như vậy, webpack đã bundle ra cho chúng ta 1 file tên là bundle.js từ 2 file hello.js và index.js. Và file index.html chỉ việc sử dụng file bundle.js đó để xuất ra kết quả trên màn hình.

         Watchmode: Watchmode sẽ giúp ta theo dõi files, và khi chúng có bất kì thay đổi nào, nó sẽ ngay lập tức build và xuất ra file output.

    • + Từ command line gõ dòng lệnh:webpack --watch
    • + Trong file config, ta sẽ thêm một key là watch và đặt nó là true
      •  module.exports = {
         entry: './index.js',
         output: { filename: 'bundle.js' },
         watch: true
       }

    Sau khi đặt watch thành true, mỗi khi chạy webpack, nó sẽ tự động rebuild file bundle mỗi khi có thay đổi.

    Gulp vs Webpack

    Giống nhau: Đều chạy trên nền Node.js và đều dùng file JS để config

    Khác nhau:

    • + Gulp: Task runner, có nhiệm vụ là compile SASS sang CSS, tối ưu hóa các tệp JS, CSS, Image...
    • + Webpack: Module bundler, có nhiệm vụ là tạo ra những file tài nguyên tĩnh từ những module cần thiết, nén file JS thành 1 file bundle.js duy nhất

         Vấn đề với Gulp là nếu code file A depend vào code ở file B, bạn cần báo cáo với Gulp để nó include file A trước, và khi bạn thêm càng lúc càng nhiều file depend chồng chéo kiểu này, cây dependency là lớn dần, và sẽ gần như không thể maintain được nữa, vì chúng ta phải config bằng tay. Trong khi đó thì webpack sẽ tự động dựng lên cây dependency và bundle ra duy nhất 1 file cho chúng ta.

         Gulp chạy lệnh để build scss ra file css nhưng Webpack sẽ nén các file vào file bundle.js mà không build file css.
