Tìm hiểu về HTML5 và CSS 3

Tran Duc Trung 18/07/2018

    1) CSS3 là gì? CSS3 có những điểm gì mới?

    - CSS là viết tắt của Cascading Style Sheet là 1 ngôn ngữ để định dạng lại các phần tử được tạo ra bởi ngôn ngữ HTML

    - CSS3 là 1 phiên bản nâng cấp của CSS, là thành phần làm nên trang web 1 cách toàn diện nhất mà không cần phải bổ sung tới các thành phần bổ sung như: JavaScript, Jquery, Flash... khiến cho trang Web tải lại nhanh chóng hơn

    - CSS3 bổ sung rất nhiều thành phần mới giúp người lập trình web giảm thiểu được rất nhiều tài nguyên từ thành phần bổ sung bên ngoài, đặc biệt CSS3 rất có lợi cho website làm việc hiệu quả mà ít tốn dung lượng tài nguyên, thời gian tải trang.

    - Các tính năng mới của CSS3:

    • + Border-radius: Tạo viền bo tròn
    • + Border-image: Tạo viền xung quanh ảnh
    • + Gradient: Tạo 1 dải màu tùy chỉnh
    • + Box-shadow: Tạo hiệu ứng bóng đổ cho element
    • + Text-shadow: Tạo hiệu ứng bóng đổ cho text
    • + Animations and Transitions: Tạo hiệu ứng chuyển động cho element thay vì phải dùng JS như trước
    • + Advance Selectors: Tăng thêm khả năng tùy chọn selectors
    • + Media query: Quy định cụ thể media hay thiết bị gì được hiển thị trong tài liệu liên quan

    2) HTML5 là gì? HTML5 có những điểm gì mới?

    - HTML (Hyper Text Markup Language): Là 1 thành phần quan trọng của website, dùng để mô tả những gì trang web hiển thị

    - HTML5 là 1 phiên bản nâng cấp của HMTL, dựa theo các tiêu chuẩn:

    • + Giảm nhu cầu cho các Plugins bên ngoài
    • + Scripting được thay thế bằng Markup bất cứ khi nào có thể
    • + Xử lý lỗi tốt hơn
    • + Được phát triển độc lập trên các thiết bị
    • + Quá trình phát triển được công khai nhằm cho người dùng và các tổ chức có thể lấy làm cơ sở

    - Các tính năng mới của HTML5: Bổ sung thêm nhiều thẻ mới

    • + <header> <footer>: Giúp người viết code có thể tách nội dung thành các block để có thể sử dụng lại nhiều lần
    • + <header> <footer>:
    • + <article>: Định nghĩa 1 phàn cụ thể của nội dung như là 1 bài viết hay 1 bình luận của người dùng
    • + <aside>: Đánh dấu nội dung bên cạnh của trang hiện tại, ví dụ như một slidebar.
    • + <nav>: Định nghĩa phần menu điều hướng cho trang
    • + <section>: Xác định các thành phần khác nhau trong website
    • + <audio> <video>: Đánh dấu những nội dung là đoạn phim hoặc âm thanh
    • + <canvas>: Cho phép bạn vẽ đồ họa mà không cần phải thông qua ngôn ngữ kịch bản nào cả (JS)
    • + <embed>: Thẻ này dùng để nhúng các nội dung hoặc các ứng dụng bên ngoài vào trang web

    3) SASS là gì? SASS vs SCSS?

    - SASS (Syntactically Awesome Style Sheet): Là 1 CSS Preprocessor - ngôn ngữ tiền xử lý CSS, nó cho phép developers viết mã bằng 1 ngôn ngữ nào đó, sau đó sẽ biên dịch nó thành CSS. Việc này sẽ giúp các developer tiết kiệm thời gian viết code và dễ dàng mở rộng, bảo trì sau này. SASS có cung cấp thêm 1 số nguyên tắc giúp quản lý CSS theo thứ tự rõ ràng và dễ dàng chỉnh sửa tái sử dụng:

    • + Biến (Variables)
    • + Quy tắc xếp chồng (Nested Rules)
    • + Quy tắc kế thừa
    • + Quy tắc mixin

    - SCSS (Sassy CSS): Cú pháp của SCSS dựa trên cú pháp của CSS, bắt đầu sử dụng {} và ; như CSS, không còn quá quan trọng việc thụt lề hay các khoảng trắng. Trong khi đó thì SASS được xem như là cú pháp thụt đầu dòng (indented syntax), tức là SASS sẽ dùng việc thụt đầu dòng để phân chia giữa các đoạn code thay vì dùng {} và ; như SCSS, điều này sẽ giúp cho code SASS ngắn gọn, dễ dàng đánh máy hơn, cách sử dụng @mixin, @include cũng được thay thế bằng ký tự : và +

    - Tuy nhiên thì việc thụt đầu dòng này sẽ làm cho việc căn chỉnh khi code đòi hỏi phải cẩn thận hơn đối với developers, bởi vì chỉ cần sai 1 thụt đầu dòng thì có thể làm hỏng toàn bộ file .sass đó. Hơn nữa với cú pháp này thì cũng sẽ gây không ít khó dễ cho việc đọc, hiểu code đối với 1 newbie

    - Còn đối với SCSS thì có cú pháp giống với CSS cho nên sẽ dễ dàng đọc code hơn. Code trong SCSS cũng tường minh hơn khi không sử dụng các ký tự để thay thế cho @mixin hay @include

    4) LESS là gì? Stylus là gì? So sánh LESS vs SASS vs Stylus

    - LESS và Stylus cũng giống như SASS là 1 CSS Preprocessor

    - Về cú pháp thì LESS gần giống với SCSS, còn Stylus thì giống với SASS, khác biệt ở chỗ là Stylus không cần ký tự : sau mỗi 1 style, việc khai báo biến của Stylus cũng không cần ký tự $ đặt trước biến

    Ví dụ:

    • + Biến (Variables):
      •  /* SASS */
        $primary-color: #333
        body
          color: $primary-color

       /* LESS */
        $primary-color: #333;
        $light-blue: $primary-color + #111
        body {
          color: $light-blue;
        }

       /* Stylus */
        primary-color: #333
        body
          color primary-color
    • + Mixins:
      •  /* SASS */
        @mixin color($n)
          background-color: $n
        div
          @include color(#333)

       /* LESS */
        .color($n) {
          background-color: $n;
        }
        div {
          .color(#333);
        }

       /* Stylus */
        color n
          background-color: n
        div
          color #333
    • + Nesting:
      •  /* SASS */
        div
          width: 200px
          p
            font-size: 17px

       /* LESS */
        Tương tự SASS

       /* Stylus */
        div
          width 200px
          p
            font-size 17px
        div
          font-size 17px
          font-weight bold
    • + @Extend:
      •  /* SASS */
        .message
          color: #333
        .success
          @extend .message
          border-color: #fff

       /* LESS */
        .message {
          color: #333;
        }
        .success {
          &:extend( .message);
          border-color: #fff;
        }

       /* Stylus */
        .message
          color #333
        .success
          @extend .message
          border-color #fff
    • * Stylus còn khác SASS ở chỗ là có thể gọi @extend từ nested selectors còn SASS thì không
      •   form
          button
            padding 10px
        a.button
          @extend form button

    5) Thế nào là responsive design? Làm thế nào để thực hiện Responsive Design?

    - Responsive design là 1 cách thức xây dựng trang web nhằm hướng đến việc giúp người dùng có những trải nghiêm tốt hơn như là dễ xem - đọc nội dung với việc thay đổi kích thước, cuộn trang web qua nhiều thiết bị (smartphone, tablet...). Nói nôm na là ứng với mỗi khung hình khác nhau trang web vẫn sẽ hiển thị ra 1 giao diện với với đầy đủ tính năng để đáp ứng được nhu cầu người dùng

    - Để có thực hiện Responsive Design thì phải cần khá nhiều thao tác, tuy nhiên có 2 bước cơ bản cần phải thực hiện:

    • + Bước 1: Thêm thẻ meta viewport vào header. Meta viewport nghĩa là một thẻ thiết lập cho trình duyệt hiển thị tương ứng với kích thước màn hình
      • <meta name="viewport" content="width=device-width, initial-scale=1">
    • + Bước 2: Viết CSS cho chiều rộng của thiết bị
    • Để viết CSS tương ứng cho chiều rộng của trình duyệt, chúng ta sẽ sử dụng thẻ truy vấn @media trong CSS3. Ví dụ:
      •   body {
          background: #fff;
          color: #333;
        }
        @media all and (max-width: 320px) {
          body {
            background: #ddd;
          }
        }

      -> Điều này có nghĩa là website mặc định sẽ có nền màu trắng, nhưng khi thu nhỏ trình duyệt xuống từ 0 - 320px thì nó sẽ có nền là màu xám.
