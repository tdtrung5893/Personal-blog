Git và những điều cơ bản

Tran Duc Trung 14/07/2018

    1. Thế nào là repository, branch?

    - Repository: Là kho lưu trữ toàn bộ thông tin của 1 project. Có 2 loại repository:

    • + Repository remote: Kho lưu trữ từ xa dùng để chia sẻ thông tin với người dùng khác
    • + Repository local: Kho lưu trữ ở máy tính của người dùng

    - Branch:

    • + Là không gian làm việc của người dùng (workspace)
    • + Trong 1 project sẽ có nhiều nhánh khác nhau và mỗi nhánh hoàn toàn độc lập với nhau.
    • + Trong quá trình làm việc có thể gộp các nhánh lại với nhau bằng lệnh rebase rồi tiếp tục làm việc bình thường.

    2. Làm thế nào để xoá một branch ở phía local, làm thế nào để xoá một branch remote?

    - Local:

      Git branch -D branch_name (xóa không cần lí do)

      hoặc

      Git branch -d branch_name (nếu đã tạo pull request và chưa được merge thì sẽ không xóa được theo cách này)

    - Remote:

    Git push remote_name -d branch_name

    3. Làm thế nào để push một branch ở local lên remote dưới một cái tên khác (Ví dụ như ở local tên branch là task#1, và muốn push lên branch task#2 ở remote)

    Git push remote_name task1:task2

    4. Thế nào là git rebase. Phân biệt rebase với merge?

    - Rebase: Tích hợp những thay đổi ở nhánh khác vào nhánh hiện tại

    Phân biệt:

    • + Rebase: Các commit của nhánh đang rebase sẽ được sao chép lại và đưa tất cả lên phía trước commit mới nhất của nhánh rebase (các commit ở nhánh đang rebase sẽ là mới nhất)
    • + Merge: Lấy tất cả những thay đổi của 2 commit mới nhất ở 2 nhánh rồi tạo thành 1 commit mới (lịch sử commit được giữ nguyên)

    5. Thế nào là git fetch. Phân biệt fetch với pull?

    - Fetch: Sau khi kéo 1 nhánh trên remote về, git sẽ không merge luôn mà sẽ lưu những thay đổi sang 1 nhánh khác

    - Pull: Sau khi kéo 1 nhánh trên remote, git sẽ tự động merge những thay đổi vào nhánh ở local

    6. Thế nào là git stash?

    - Dùng để lưu những thay đổi khi muốn chuyển sang 1 branch khác mà không muốn commit

    7. Làm thế nào xoá bỏ trạng thái của một vài commit gần đây?

    Git reset --hard HEAD~n (n là các số tự nhiên như 1,2,3,4...)

    8. Làm thế nào để gộp một vài commit thành 1 commit duy nhất?

    Git rebase -i HEAD~n (n là các số tự nhiên như 1,2,3,4...)

    9. Phân biệt git reset, git reset --hard, git reset --soft?

    Git reset hoặc Git reset HEAD~n: Di chuyển con trỏ HEAD về commit reset, giữ nguyên thay đổi các file và loại bỏ những thay đổi ở stage
    Git reset --hard HEAD~n: Di chuyển con trỏ HEAD về commit reset, giữ nguyên thay đổi các file và loại bỏ những thay đổi ở stage
    Git reset --soft HEAD~n: Di chuyển con trỏ HEAD về commit reset và giữ nguyên tất cả những thay đổi của các file kể cả ở stage

    10. Thế nào là cherry-pick, khi nào thì dùng cherry-pick?

    Cherry-pick: Dùng để checkout commit ở nhánh khác vào nhánh hiện tại

    Ví dụ ở nhánh develop đang có commit mới nhất là "update user" và người dùng hiện tại đang làm việc trên nhánh delete_user

    Khi muốn lấy hết tất cả những thay đổi trong commit "update user" ở nhánh develop sang delete_user thì lúc này người dùng có thể sử dụng:

    Git cherry-pick develop

    11. Git flow

    • 1. Fork
    • 2. Clone
    • 3. Git remote add
    • 4. Git checkout -b branch_name
    • 5. Git add/commit
    • 6. Git rebase *
    • 6.1. Git checkout master
    • 6.2. Git pull
    • 6.3. Git checkout branch_name
    • 6.4. Git rebase master
    • 7. Git push origin branch_name
    • 8. Create pull request
