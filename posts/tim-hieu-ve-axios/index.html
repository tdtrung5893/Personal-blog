Tìm hiểu về Axios

Tran Duc Trung 01/08/2018

    Lodash là gì?

    - Axios là 1 HTTP Client được viết dựa trên Promises được dùng để hỗ trợ cho việc xây dựng các ứng dụng API và có thể được sử dụng ở cả trình duyệt hay Node.js

    - Cài đặt:

     npm install axios --save-dev

    - Các Request có thể được tạo ra bằng cách chỉ định các config thích hợp cho axios. 1 số option phổ biến:

    •    + url: Là đích đến của Request và là 1 đường dẫn tương đối
    •    + method: Là phương thức được sử dụng để thực hiện Request, mặc định trong axios là 'GET'
    •    + baseurl: Sẽ được gắn vào trước url vì url là đường dẫn tương đối
    •    + data: Là dữ liệu kiểu object sẽ được gửi theo body của Request, chỉ khả dụng đối với các phương thức POST, PUT, PATCH
    •    + header: Là 1 object bao gồm các cặp key - value có thể gửi trong header của Request
    •    + params: Là các tham số được gửi lên bằng cách đính vào URL dưới dạng query string. Giá trị của params là 1 object bao gồm các cặp key - value
    •    + Response Type: Chỉ định kiểu dữ liệu mà Server sẽ trả về

    - Thực hiện Request với phương thức GET

     axios({
       url: '/user',
       method: 'GET',
       params: {
         id: 1
       }
     })
     .then(function(response){
       console.log(response);
     })
     .catch(function(err){
       console.log(err);
     })

    - Alias: Là tập hợp những hàm rút gọn để thực hiện các kiểu Request khác nhau

    • + Các hàm GET, DELETE, OPTIONS, HEAD đều nhận vào 2 tham số là URL và 1 object options
    • + Các hàm POST, PUT, PATCH ngoài 2 tham số giống với các hàm trên thì còn nhận thêm 1 tham só là 1 object dữ liệu
     axios.post({'/login',
       {
         username: 'tdtrung_93',
         email: 'tdtrung5893@gmail.com'
       },
       { options }
     })
     .then(function(response){
       console.log(response);
     })
     .catch(function(err){
       console.log(err);
     })

    - Khi tạo ra 1 Request thì Axios sẽ trả về 1 Promises có trạng thái là resolve và dữ liệu trả về sẽ gồm có các thuộc tính sau:

    • + Data: Là dữ liệu được trả về bởi Server
    • + Status: Là mã HTTP Status trả về từ Server
    • + Status Text: Là message của HTTP Status
    • + Header: Tất cả các header được gửi về từ Server
    • + Config: Là các cấu hình khi thực hiện Request
    • + Request: Là Request thực hiện để nhận được Response này
