Thế nào là Prototype

Tran Duc Trung 21/07/2018

    Thế nào là prototype?

    - Trong JavaScript thì mọi thứ đều là Object (ngoại trừ Undefine và Null) và Object function này sẽ có 1 thuộc tính đó là prototype, nó đơn giản là trỏ đến 1 object khác

    - Object.prototype là 1 object và nó chứa khá nhiều các function thuộc tính mà chúng ta vẫn dùng như toString(), valueOf(), ... Nó còn có 1 property là constructor mà trỏ ngược về lại Object() function

       function foo(name) {
         this.myname = name;
       }

    - Khi chạy đoạn code trên thì JavaScript sẽ tự động tạo ra thuộc tính prototype cho function foo() và trỏ đến 1 anonymous object và anonymous này có 1 constructor trỏ ngược về function foo(). Đồng thời foo.prototype cũng trỏ đến object.prototype thông qua [[prototype]]

    - Thêm phương thức cho function foo() bằng prototype:

       foo.prototype.yourname = function() {
         alert('I am ' + this.myname);
       };
       var a = new foo('Trung');
       a.yourname();    // I am Trung

    -> Khi chạy câu lệnh trên thì Javascript engine sẽ tìm kiếm hàm yourname ở object a đầu tiên, nếu không có thì nó sẽ tìm đến tiếp foo.prototype object, khi chưa tìm thấy thì sẽ tiếp tục tìm hết các object trong chuỗi prototype chain. Nếu vẫn không tìm thấy thì sẽ báo lỗi.

    - Thêm phương thức cho đối tượng:

       var b = new foo('Duc');
       b.say = function() {
         alert(this.yourname + ' say hello');
       };
       b.say();    // I am Duc say hello

    -> say() chỉ có ở object b nên chỉ object b mới gọi được phương thức này.

    - Shadowing:

       a.yourname = function() {
         alert('My name is ' + this.myname);
       };

       a.yourname();    // My name is Trung

    -> Do phương thức yourname có ở cả object a và function foo() cho nên phương thức yourname ở function foo() đã bị che mờ đi.
