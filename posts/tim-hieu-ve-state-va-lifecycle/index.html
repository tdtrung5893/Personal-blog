Tìm hiểu về State và LifeCycle

Tran Duc Trung 05/08/2018

    State là gì?

    - Trong Reactjs, State được dùng để biểu diễn trạng thái của Component.

    - State là Private và chỉ có thể thay đổi ở bên trong của chính Component đó

    - Để khởi tạo State ta dùng lệnhthis.state.Ví dụ:

     import React, {Component} from 'react';
     class App extends Component {
       constructor(props){
         super(props)
         this.state = {
           name: "Tran Duc Trung"    // khởi tạo state bằng cách thêm vào 1 object
         }
       }
       render(){
         return(<h1&glt;{ this.state.name }</h1>)    // gọi đến state this.state.name
       }
     }

    - Để thay đổi giá trị của State ta dùng lệnhthis.setState.Ví dụ:

     import React, {Component} from 'react';
     class App extends Component {
       constructor(props){
         super(props)
         this.state = {
           name: "Tran Duc Trung"
         }
         this.handleChangeState = this.handleChangeState.bind(this)
       }
       handleChangeState(){
         this.setState({
           name: "Hello tdtrung93"
         })
       }
       render(){
         return(
           <h1>{ this.state.name }</h1>
           <button onClick={this.handleChangeState}>Change State</button>
         )
       }
     }

    - Đoạn code trên sẽ render ra 1 nút button "Change State" và khi Click vào button đó thì trạng thái name của Component App sẽ thay đổi và hàm render sẽ được render lại

    Thế nào là LifeCyle?

    - Lifecycle trong ReactJS dùng để chỉ vòng đời của 1 Component, thực chất là nó bao gồm những method được gọi là Lifecycle methods, đây là những method mà ta có thể gọi tại 1 thời điểm nào đó trong vòng đời của 1 Component. Có 3 loại lifecycle method là Mounting, Updating, Unmounting.

    • + Mounting: Đây là những method được gọi trong lần Component được render lần đầu tiên. Có 4 method Mounting lifecycle method là:
      • Constructor(): Đây là method đầu tiên được gọi khi khởi tạo Component, chúng ta có thể thiết lập State và ràng buộc các class methods ở trong này
      • ComponentWillMount(): Method này được gọi trước khi phương thức render được gọi lần đầu tiên vì vậy method này chỉ xuất hiện 1 lần duy nhất, những lần render sau method này sẽ không được gọi nữa
      • Render(): Method này là bắt buộc, nó nhận vào các state, props và trả ra các element
      • ComponentDidMount(): Method này được thực thi khi 1 component được render. Đây là thời điểm thích hợp nhất để thực hiện việc giao tiếp với server để lấy dữ liệu từ API.
    • + Updating: Các lifecycle methods của Updating sẽ được gọi bắt đầu với lần render thứ hai. Có 5 method Updating lifecycle method là:
      • ComponentWillReceiveProps(nextProps): Method này sẽ được gọi khi thuộc tính props được update và trước khi component được render lại
      • ShouldComponentUpdate(nextProps, nextState): Method này sẽ return kết quả về true hoặc false, nếu false thì tất cả những method phía sau sẽ không được thực hiện
      • ComponentWillUpdate(nextProps, nextState): Được gọi khi chúng ta update state của component trước khi nó render lại.
      • Render()
      • ComponentDidUpdate(prevProps, prevState): Sau khi render lại những thay đổi xong thì method này sẽ được gọi
    • + Unmounting: Lifecycle chỉ có 1 method duy nhất là componentWillUnmount() và nó được gọi khi chúng ta unmout 1 component kiểu như xóa nó khỏi react.

    - Ví dụ về thứ tự được gọi của lifecycle methods:

     import React, {Component} from 'react';
     class LifeCycle extends Component {
       constructor(props){
         super(props)
         this.state = {
           title: "Content",
         }
         this.handleChangeContent = this.handleChangeContent.bind(this)
         console.log("constructor");
       }

       handleChangeContent(){
         this.setState({
           title: "Changed Content"
         })
       }

       render(){
         return(
           <div className="card">
             <div className="card-header">ReacJS</div>
             <div className="card-block">
               <h4 className="card-title">Special title treatment</h4>
               <Post content = { this.state.title } className="card-text"></Post>
               <button onClick={this.handleChangeContent} type="button" className="btn btn-primary">Change content</button>
             </div>
           </div>
         );
       }
     }

     class Post extends Component {
       componentWillMount(){
         console.log("componentWillMount");
       }

       componentDidMount(){
         console.log("componentDidMount");
       }
       componentWillReceiveProps(newProps) {
         console.log("componentWillReceiveProps")
       }

       shouldComponentUpdate(nextProps, nextState){
         console.log("shouldComponentUpdate");
         return true;
       }
       componentWillUpdate(){
         console.log("componentWillUpdate");
       }

       componentDidUpdate(){
         console.log("componentDidUpdate");
       }

       render(){
         console.log("render")
         return(
           <div>
             <h3>{this.props.content}</h3>
           </div>
         );
       }
     }

     export default LifeCycle;

    Kết quả render lần đầu tiên sẽ lần lượt là:

    • constructor
    • componentWillMount
    • render
    • componentDidMount

    Sau khi nhấn nút Change content thì quá trình update sẽ diễn ra và các lifecycle khác sẽ thực hiện theo thứ tự:

    • componentWillRecevieProps
    • shouldComponentUpdate
    • componentWillUpdate
    • render
    • componentDidUpdate
