Tìm hiểu về Presentational/Container Components

Tran Duc Trung 12/08/2018

    Container Components

    - Container Components: Là những component mà nhiệm vụ chính của nó là liên kết với Store của project và fetching dữ liệu. Sau khi đã fetching dữ liệu thì các container component sẽ có những xử lý logic và truyền kết quả cuối cùng tới những component thuần hay còn gọi là Presentational Component.

    Presentational Component

    - Presentational Component: Là những component mà nhiệm vụ chính của nó là hiển thị dữ liệu ra màn hình và tại đây sẽ không có bất cứ các xử lý logic nào bên trong cả.

    - Ví dụ: Ta có 1 Container Component dùng để kéo dữ liệu từ Store

     function PostList({ newPosts }) {
       return (
         <div>
           {newPosts.map(post => {
             return (
               <Post post={ post } key={ post.id } />
             );
           })}
         </div>
       );
     }

     function mapStateToProps(state){
       return {
         newPosts: state.posts
       };
     };

     export default connect(mapStateToProps, null)(PostList);

    - và 1 Presentational component dùng để hiển thị dữ liệu

     class Post extends Component{
       render(){
         let { post } = this.props
         return(
           <div>
             <h2>{ post.title }</h2>
             <p>{ post.body }</p>
           </div>
         
       }
     }

    - Lợi ích chính của việc phân tách presentation component và contatiner chính là nâng cao được khả năng tái sử dụng các component. Ngoài ra thì nó còn giúp App của chúng ta trông clean hơn, dễ maintain hơn và tách biệt giữa phần xử lý logic và phần view.
