Scope và Closure trong JavaScript

Tran Duc Trung 18/07/2018

    Thế nào là scopes ? Trong Javascript có những kiểu scopes gì?

    - Scope: Là phạm vị truy cập, có thể sử dụng được các variable hoặc function

    - Các loại scope trong JavaScript:

    • + Global scope (scope toàn cục): Có nghĩa là có thể truy cập từ bất cứ đâu trong chương trình
      •  var name = "Trung";
       function myfunc() {
         alert(name);
       }
       myfunc();    // Trung
       alert(name);    // Trung
       Biến name có phạm vị truy cập là global scope nên nó có thể truy cập được ở trong lẫn bên ngoài function
    • + Local Scope (scope cục bộ): Chỉ có thể truy cập được từ 1 phạm vi nhất định
      •  function myfunc() {
         var name = "Trung";
         alert(name);
         }
       myfunc();    // Trung
       alert(name);    // Undefined
       Biến name có phạm vị truy cập là local scope nên nó không thể truy cập được từ bên ngoài function, do đó dẫn đến Undefined
    • + Function scope: Là scope được tạo ra chỉ cho function đó sử dụng. Nó chính là mọi thứ nằm trong dấu {} của hàm.
      •  var myfunc = function() {
       // code here
       }

    Thế nào là closure?

    - Closure: Là một hàm được tạo ra từ bên trong một hàm khác (hàm cha), nó có thể sử dụng các biến toàn cục, biến cục bộ của hàm cha và biến cục bộ của chính nó. Việc viết hàm theo kiểu closure trong một số trường hợp sẽ giúp code nhìn sáng và dễ quản lý hơn, linh hoạt hơn trong việc xử lý các chức năng.

       function showmessage(message) {    // Tạo hàm closure
         return function() {
           alert(message + ' ' + 'Trung');
         };
       }
       var funcmessage = showmessage('Hello');    // Khởi tạo hàm closure
       funcmessage();    // Hello Trung

    - Closure có thể thay đổi giá trị biến toàn cục lẫn cục bộ: Các closure có thể sử dụng biến toàn cục, biến cục bộ của hàm cha và biến cục bộ của chính nó. Không chỉ sử dụng được mà còn có khả năng thay đổi giá trị của các biến đó Kỹ thuật composition sẽ làm cho code dài hơn nhưng lại clear hơn và dễ dàng thay đổi, bảo trì sau này. Ngoài ra nó còn làm linh hoạt hơn trong cách sử dụng đối với các class mới tạo.

     function person() {
       var name = ' ';
       return {
         set: function(inname) {
           name = inname;
         },
         getname: function() {
           return name;
         }
       };
     }
     var object = person();
     object.set('Trung');
     alert(object.getname());    // Trung

    - Độ ưu tiên các biến: Như ta biết closure function có thể sử dụng biến tại ba vị trí đó là biến toàn cục, biến cục bộ hàm cha và biến cục bộ của chính nó. Bây giờ có một trường hợp là tên các biến ở ba vị trí đó bị trùng nhau. Và trong trường hợp này thì nó sẽ ưu tiên từ trong ra ngoài.

     var message = 'Biến toàn cục';
     function showMessage() {
       var message = 'Biến cục bộ của hàm cha';
       return function(){
         var message = 'Biến cục bộ của hàm con';
         alert(message);
       };
     }
     var messageFunc = showMessage();
     messageFunc();    // Biến toàn cục của hàm con
