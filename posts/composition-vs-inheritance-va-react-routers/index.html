Composition vs Inheritance và React Routers

Tran Duc Trung 10/08/2018

    Composition vs Inheritance

    - Trong React, thuộc tính props của Component có thể được truyền vào các giá trị bất kỳ như là: number, string, object, React Element, thậm chí là cả 1 function... Vì vậy ta có thể cấu hình 1 Component với thuộc tính props của nó => Kỹ thuật này được gọi là Composition

    - Việc sử dụng Composition thay vì Inheritance sẽ giúp chúng ta có thể tái sử dụng code giữa các Component

     class App extends Component {
       constructor(props) {
         super(props)
       }

       render(){
         return (
           <div>
             {this.props.up}
             {this.props.children}
           </div>
         );
       }
     }

     function UpComponent(props) {
       return(
         <div>This is Up Component</div>
       )
     }

     ReactDOM.render(
       <App up={}>
         <h1>This is Children</h1>
       </App>
       , document.getElementById('root')
     );

    React Routers

    - Trong React, chỉ có 1 file HTML được gọi. Bất cứ khi nào người dùng nhập 1 đường dẫn mới, thay vì lấy dữ liệu từ server, React Router sẽ chuyển sang 1 Component khác ứng với mỗi đường dẫn mới. Đối với người dùng thì họ đang chuyển qua lại giữa nhiều trang nhưng thực tế, mỗi Component đơn lẻ sẽ được render lại phụ thuộc đường dẫn.

    - Cài đặt React Routers:

     npm install react-router-dom --save

    - Import Component "BrowerRouter" từ thư viện "react-router-dom" cùng với "Link" và "Route".

     import {BrowserRouter, Route, Link} from 'react-router-dom';

    - BrowserRouter: Các Router chỉ có thể có 1 thành phần con đơn lẻ, vì vậy ta sẽ dùng thẻ <BrowserRouter> để gói Component ngoài cùng. Ví dụ:

     ReactDOM.render((
       <BrowserRouter>
         <App/>
       </BrowserRouter>
     ), document.getElementById('root'));

    - Link: Link được sử dụng để điều hướng bên trong các route. Nó tương đương với thẻ trong html và với thẻ link_to trong Rails.

     <ul>
       <li><Link to="/">Home</Link></li>
       <li><Link to="/about">About</Link></li>
       <li><Link to="/contact">Contact</Link></li>
     </ul>

    - Route: Có nhiệm vụ render UI khi người dùng truyền vào 1 location trùng với path được định nghĩa trong Route

     <div>
       <Route exact path="/" component={Home} />
       <Route path="/about" component={About}/>
       <Route path="/contact" component={Contact}/>
     </div>
    • + exact: Khi pathname là "/" thì nó sẽ khớp với cả "/", "/about", "/contact", vì vậy để nó chỉ khớp với đúng "/" thì ta sử dụng exact
    • + Hoặc nếu không muốn sử dụng exact thì ta có thể dùng Switch để thay thế. Switch được dùng để nhóm các Route lại với nhau, Switch sẽ lặp qua các phần tử con của nó (routes) và chỉ render phần tử đầu tiên phù hợp với tên đường dẫn hiện tại.

    - Ví dụ:

     <Switch>
       <Route path="/" component={Home} />
       <Route path="/about" component={About}/>
       <Route path="/contact" component={Contact}/>
     </Switch>

    - Nested Route: Ví dụ có 1 danh sách các bài viết và ta cần hiển thị chi tiết các bài viết đó thì lúc này ta sẽ sử dụng Nested Route

     <Switch>
       <Route path='/posts/:id' component={Posts}>
         <Route path="post" component={PostDetails} />
       </Route>
     </Switch>
